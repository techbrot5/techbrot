4 · Gates (every build)

Section variety

  • #12a ≤ 1 numbered system per page
  • #12b no two adjacent section--* alike (read the declared sequence)
  • #12c variety floor on long pages
  • #12d hero fits the type (legal/utility ≠ marketing)

Honesty (trust = primary asset)

  • real proof only · no fabricated stats/counters/urgency
  • 2 real Clutch reviews · no AggregateRating until 5+
  • real Intuit ProAdvisor badge artwork only (none invented here)
  • independent firm · not Intuit · honest triage

Forms ruling

Full forms only on /contact/ + /file-review/; minimal 4–5 field form at the bottom of state pillars. Everything else converts by CALL.

6 · Capture vs. wire

CODE CAPTURES AS-IS
  • Equity guardrail. The re-skin applies the new system to EXISTING pages WITHOUT altering heading text, FAQ content, intent, or URLs — restyle only, never rewrite content/equity; Equity-GREEN vs baseline.json must hold.
  • All markup, inline styles → tokens (re-cut the 10 CSS layers to these values; css-drift gate green)
  • The section-layout library → Nunjucks macros + section--* classes
  • The 4 signature diagrams + 24 custom icons + isometric piece (live SVG, parameterized)
  • Vanilla nav JS (mega/drawer/accordion: data-open toggle, aria-expanded, Esc/outside-close) → tb-nav.js
  • Scroll-reveal + draw-on (CSS), reduced-motion zeroing, TOC scroll-spy (IntersectionObserver)
  • All JSON-LD blocks, speakable / QAPage / FAQPage / Breadcrumb / Review nodes, semantic structure
  • Every component state (forms incl. loading/success/error) + utility states
CODE WIRES
  • Form submission handlers (Pages Functions) + email delivery + honeypot enforcement
  • Origin/intent passing: URL param / referrer → [data-origin-*] H1 + hidden fields + pre-fill
  • GA4 events: phone_click on every tel: ([data-ga]) + lead_submit on success
  • Phone — MUST swap before production. Every tel: is +1-XXX-XXX-XXXX (display (XXX) XXX-XXXX) — an obvious placeholder, never a real line. Code swaps the real founder number centrally; the XXX placeholder must never reach a production build.
  • Per-page front-matter: type + ordered sections: + data arrays
  • Real content/figures per page (replace illustrative diagram data with the page's real numbers)
  • OG-image template (per type): mono headline + tie-out mark on cold field, 1200×630 — render at build from page title + type; on every money/client page
